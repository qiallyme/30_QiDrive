# Title 10. QiLabs

## Citation

- Legal citation: `QiCode Title 10`
- Article citation: `QiCode Title 10, Article #`
- Section citation: `QiCode Sec. 10.##.###`
- Stable ID alias: `QiCode T10.A##.S###.L###`

## Rule of Construction

- `Rule 10.001.L001` This title governs `C:\QiLabs` as the full root operating environment. (stable ID `T10.R001.L001`)
- `Rule 10.001.L002` A section states a governing principle; a line states a citeable rule or application. (stable ID `T10.R001.L002`)
- `Rule 10.001.L003` Exceptions must be named, reviewed, and linked to the record that justifies them. (stable ID `T10.R001.L003`)

## Article 1. Root Authority

Stable ID: `T10.A01`

### Sec. 10.01.010. Primary Function

Stable ID: `T10.A01.S010`

- `Line 1` QiLabs is the root operating environment for all systems, applications, vaults, and projects under `C:\QiLabs`. (`Sec. 10.01.010.L001`; stable ID `T10.A01.S010.L001`)
- `Line 2` QiLabs must not be used as a dumping ground for unrelated material. (`Sec. 10.01.010.L002`; stable ID `T10.A01.S010.L002`)
- `Line 3` If use is ambiguous, the placement rule for the nearest parent title controls. (`Sec. 10.01.010.L003`; stable ID `T10.A01.S010.L003`)

### Sec. 10.01.020. Root Folder Authority

Stable ID: `T10.A01.S020`

- `Line 1` Every top-level folder under `C:\QiLabs` must have one clear, documented purpose governed by a QiCode title. (`Sec. 10.01.020.L001`; stable ID `T10.A01.S020.L001`)
- `Line 2` Root folders are governed domains, not casual categories. (`Sec. 10.01.020.L002`; stable ID `T10.A01.S020.L002`)
- `Line 3` No new root folder may be created without a corresponding doctrine update in QiCode. (`Sec. 10.01.020.L003`; stable ID `T10.A01.S020.L003`)

### Sec. 10.01.030. QiCode Coverage

Stable ID: `T10.A01.S030`

- `Line 1` QiCode Titles 10–18 define the developer and system doctrine for the QiLabs root. (`Sec. 10.01.030.L001`; stable ID `T10.A01.S030.L001`)
- `Line 2` Each title corresponds to exactly one top-level folder or system domain. (`Sec. 10.01.030.L002`; stable ID `T10.A01.S030.L002`)
- `Line 3` Title 10 is the constitution for the entire root; all other titles are subordinate. (`Sec. 10.01.030.L003`; stable ID `T10.A01.S030.L003`)

## Article 2. System Map

Stable ID: `T10.A02`

### Sec. 10.02.010. Current Root Domains

Stable ID: `T10.A02.S010`

- `Line 1` `00_QiLabs.workspace` — workspace tools, templates, housekeeping, and manifests (Title 11). (`Sec. 10.02.010.L001`; stable ID `T10.A02.S010.L001`)
- `Line 2` `10_QiSpark` — docs, landing, doctrine, and navigation layer (Title 12). (`Sec. 10.02.010.L002`; stable ID `T10.A02.S010.L002`)
- `Line 3` `20_QiServer` — runtime, infrastructure, and services (Title 13). (`Sec. 10.02.010.L003`; stable ID `T10.A02.S010.L003`)

### Sec. 10.02.020. Additional Root Domains

Stable ID: `T10.A02.S020`

- `Line 1` `25_QiWorkers` — background jobs, APIs, sync, and automation workers (Title 14). (`Sec. 10.02.020.L001`; stable ID `T10.A02.S020.L001`)
- `Line 2` `30_QiDrive` — file and markdown source-of-truth layer (Title 15). (`Sec. 10.02.020.L002`; stable ID `T10.A02.S020.L002`)
- `Line 3` `40_QiVault` — markdown vault and local knowledge workspace (Title 16). (`Sec. 10.02.020.L003`; stable ID `T10.A02.S020.L003`)

### Sec. 10.02.030. App and Site Domains

Stable ID: `T10.A02.S030`

- `Line 1` `60_QiApps` — app projects and UI/tooling layers (Title 17). (`Sec. 10.02.030.L001`; stable ID `T10.A02.S030.L001`)
- `Line 2` `70_QiSites` — public and static site output (Title 18). (`Sec. 10.02.030.L002`; stable ID `T10.A02.S030.L002`)
- `Line 3` Any folder not listed above must be documented and assigned to a title before active use. (`Sec. 10.02.030.L003`; stable ID `T10.A02.S030.L003`)

## Article 3. Source of Truth

Stable ID: `T10.A03`

### Sec. 10.03.010. Canonical Authority

Stable ID: `T10.A03.S010`

- `Line 1` QiCode is the canonical governance layer; QiLabs root folders are its governed objects. (`Sec. 10.03.010.L001`; stable ID `T10.A03.S010.L001`)
- `Line 2` When a folder's purpose is unclear, the QiCode title for that folder controls. (`Sec. 10.03.010.L002`; stable ID `T10.A03.S010.L002`)
- `Line 3` Conflicts between folder usage and title doctrine must be resolved through a doctrine update, not a workaround. (`Sec. 10.03.010.L003`; stable ID `T10.A03.S010.L003`)

## Article 4. Root Folder Boundaries

Stable ID: `T10.A04`

### Sec. 10.04.010. Single Domain Rule

Stable ID: `T10.A04.S010`

- `Line 1` Each root folder governs exactly one domain; cross-domain material must be placed in the owning domain. (`Sec. 10.04.010.L001`; stable ID `T10.A04.S010.L001`)
- `Line 2` Shared dependencies must be documented with source mappings in the relevant title. (`Sec. 10.04.010.L002`; stable ID `T10.A04.S010.L002`)
- `Line 3` Boundary disputes are resolved by reviewing the primary function in the domain's QiCode title. (`Sec. 10.04.010.L003`; stable ID `T10.A04.S010.L003`)

## Article 5. Naming and Numbering

Stable ID: `T10.A05`

### Sec. 10.05.010. Folder Numbering Convention

Stable ID: `T10.A05.S010`

- `Line 1` Root folders use two-digit prefixes (`00_`, `10_`, `20_`, etc.) to define band and order. (`Sec. 10.05.010.L001`; stable ID `T10.A05.S010.L001`)
- `Line 2` Band numbers are reserved; do not insert folders between bands without a QiCode update. (`Sec. 10.05.010.L002`; stable ID `T10.A05.S010.L002`)
- `Line 3` Names within a folder must follow the parent title's naming conventions. (`Sec. 10.05.010.L003`; stable ID `T10.A05.S010.L003`)

## Article 6. Lifecycle and Archive Rules

Stable ID: `T10.A06`

### Sec. 10.06.010. Archive Policy

Stable ID: `T10.A06.S010`

- `Line 1` Retired or superseded material must be moved to a clearly labeled archive folder within the same domain. (`Sec. 10.06.010.L001`; stable ID `T10.A06.S010.L001`)
- `Line 2` Archived content must retain its original frontmatter and a note explaining why it was archived. (`Sec. 10.06.010.L002`; stable ID `T10.A06.S010.L002`)
- `Line 3` Do not delete root domain folders; update the governing title instead and archive subdirectories as needed. (`Sec. 10.06.010.L003`; stable ID `T10.A06.S010.L003`)
