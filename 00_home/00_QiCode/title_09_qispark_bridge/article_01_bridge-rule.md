## Article 1. Bridge Rule

Stable ID: `T09.A01`

### Sec. 9.01.010. Personal First

Stable ID: `T09.A01.S010`

- `Line 1` QiSpark technical doctrine serves QiCode; it does not replace the personal life doctrine. (`Sec. 9.01.010.L001`; stable ID `T09.A01.S010.L001`)
- `Line 2` Apply this provision with truth, care, consent, and proportion. (`Sec. 9.01.010.L002`; stable ID `T09.A01.S010.L002`)
- `Line 3` This bridge continues into developer and system doctrine at QiCode Titles 10â€“18, which govern QiLabs root domains. (`Sec. 9.01.010.L003`; stable ID `T09.A01.S010.L003`)

### Sec. 9.01.020. Translation

Stable ID: `T09.A01.S020`

- `Line 1` Technical systems translate doctrine into infrastructure, data, workflows, and tools. (`Sec. 9.01.020.L001`; stable ID `T09.A01.S020.L001`)
- `Line 2` Apply this provision with truth, care, consent, and proportion. (`Sec. 9.01.020.L002`; stable ID `T09.A01.S020.L002`)
- `Line 3` If the provision is used in a project, link back to the record, output, or decision that applies it. (`Sec. 9.01.020.L003`; stable ID `T09.A01.S020.L003`)

### Sec. 9.01.030. Subordination

Stable ID: `T09.A01.S030`

- `Line 1` When technical elegance conflicts with humane use, humane use governs unless safety requires otherwise. (`Sec. 9.01.030.L001`; stable ID `T09.A01.S030.L001`)
- `Line 2` Apply this provision with truth, care, consent, and proportion. (`Sec. 9.01.030.L002`; stable ID `T09.A01.S030.L002`)
- `Line 3` If the provision is used in a project, link back to the record, output, or decision that applies it. (`Sec. 9.01.030.L003`; stable ID `T09.A01.S030.L003`)
