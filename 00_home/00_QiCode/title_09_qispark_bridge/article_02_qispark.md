## Article 2. QiSpark

Stable ID: `T09.A02`

### Sec. 9.02.010. Dev Doctrine

Stable ID: `T09.A02.S010`

- `Line 1` QiSpark holds the technical development doctrine: standards, structure, registries, schemas, decisions, and templates. (`Sec. 9.02.010.L001`; stable ID `T09.A02.S010.L001`)
- `Line 2` Apply this provision with truth, care, consent, and proportion. (`Sec. 9.02.010.L002`; stable ID `T09.A02.S010.L002`)
- `Line 3` If the provision is used in a project, link back to the record, output, or decision that applies it. (`Sec. 9.02.010.L003`; stable ID `T09.A02.S010.L003`)

### Sec. 9.02.020. System Layers

Stable ID: `T09.A02.S020`

- `Line 1` QiSpark feeds QiServer, QiVault, QiApps, and related implementation layers. (`Sec. 9.02.020.L001`; stable ID `T09.A02.S020.L001`)
- `Line 2` Apply this provision with truth, care, consent, and proportion. (`Sec. 9.02.020.L002`; stable ID `T09.A02.S020.L002`)
- `Line 3` If the provision is used in a project, link back to the record, output, or decision that applies it. (`Sec. 9.02.020.L003`; stable ID `T09.A02.S020.L003`)

### Sec. 9.02.030. Engineering Record

Stable ID: `T09.A02.S030`

- `Line 1` Technical decisions belong in ADRs, standards, registries, and system documentation. (`Sec. 9.02.030.L001`; stable ID `T09.A02.S030.L001`)
- `Line 2` Apply this provision with truth, care, consent, and proportion. (`Sec. 9.02.030.L002`; stable ID `T09.A02.S030.L002`)
- `Line 3` If the provision is used in a project, link back to the record, output, or decision that applies it. (`Sec. 9.02.030.L003`; stable ID `T09.A02.S030.L003`)
