> # Best way to avoid repeats
> Use **data validation or Apps Script on `client_id` column** so duplicates get blocked.
> 
> ## Apps Script to reject duplicate client IDs
> This is the useful part.
> 
> ```javascript
> function onEdit(e) {
>   const sheetName = "Client_ID_Registry";
>   const col = 1; // client_id in column A
>   const row = e.range.getRow();
>   const column = e.range.getColumn();
>   const sheet = e.range.getSheet();
> 
>   if (sheet.getName() !== sheetName || column !== col || row === 1) return;
> 
>   let value = String(e.range.getValue()).toUpperCase().trim();
>   if (!value) return;
> 
>   e.range.setValue(value);
> 
>   const formatRegex = /^[A-Z]{3}\d{4}$/;
>   if (!formatRegex.test(value)) {
>     SpreadsheetApp.getUi().alert("Invalid client ID. Use 3 letters + 4 digits, like BBR4821.");
>     e.range.clearContent();
>     return;
>   }
> 
>   const values = sheet.getRange(2, 1, sheet.getLastRow() - 1, 1).getValues().flat();
>   const matches = values.filter(v => String(v).toUpperCase().trim() === value).length;
> 
>   if (matches > 1) {
>     SpreadsheetApp.getUi().alert("Duplicate client ID: " + value);
>     e.range.clearContent();
>   }
> }
> ```
> 
> That gives you:
> - uppercase normalization
> - format enforcement
> - no duplicates

> # What to stop doing
> Do **not** build a system that calculates “next BBR number” unless you want multiple separate IDs under the same stem pool.
> 
> You don’t.
> 
> You want:
> - one client
> - one ID
> - many files using that one ID
> 
> That’s a registry problem, not a sequence problem.
