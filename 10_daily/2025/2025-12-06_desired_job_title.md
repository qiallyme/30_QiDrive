> ## 3. Make the mapper smarter: use question context
> 
> Right now, your `FieldDescriptor` is probably something like:
> 
> ```ts
> export interface FieldDescriptor {
>   nameAttr?: string;
>   idAttr?: string;
>   labelText?: string;
>   placeholder?: string;
>   ariaLabel?: string;
>   elementHandle: ElementHandle;
> }
> ```
> 
> You want to extend it with **contextText** and **fieldKind**:
> 
> ```ts
> export type FieldKind = 'text' | 'textarea' | 'select' | 'radio' | 'checkbox';
> 
> export interface FieldDescriptor {
>   nameAttr?: string;
>   idAttr?: string;
>   labelText?: string;
>   placeholder?: string;
>   ariaLabel?: string;
>   contextText?: string;   // nearby question / prompt text
>   fieldKind: FieldKind;
>   elementHandle: ElementHandle;
> }
> ```
> 
> ### a) Build `contextText`
> 
> In `fieldMapper.ts`, when you collect fields, do something like (pseudo-ish):
> 
> ```ts
> const descriptors: FieldDescriptor[] = await page.evaluateHandle(/* ... */)
> ```
> 
> But inside the evaluation, for each element:
> 
> - Find the nearest `<label>`  
> - AND look at **previous siblings / parent text** (like the `<p>` or `<div>` that says the full question).
> 
> Rough idea inside `page.evaluate`:
> 
> ```ts
> function getContextText(el: HTMLElement): string {
>   const label = el.closest('label') || document.querySelector(`label[for="${el.id}"]`);
>   const labelText = label?.innerText || '';
> 
>   // also grab text from the closest question container
>   const container = el.closest('div, section, li') || el.parentElement;
>   const containerText = container?.innerText || '';
> 
>   return (labelText + ' ' + containerText).toLowerCase().replace(/\s+/g, ' ').trim();
> }
> ```
> 
> Then you set `contextText` for each descriptor.
> 
> ### b) Detect field kind
> 
> Still inside that evaluator, set:
> 
> ```ts
> let fieldKind: FieldKind = 'text';
> if (el.tagName === 'SELECT') fieldKind = 'select';
> else if (el.tagName === 'TEXTAREA') fieldKind = 'textarea';
> else if (el.tagName === 'INPUT') {
>   const type = (el.getAttribute('type') || 'text').toLowerCase();
>   if (['radio'].includes(type)) fieldKind = 'radio';
>   else if (['checkbox'].includes(type)) fieldKind = 'checkbox';
>   else fieldKind = 'text';
> }
> ```
> 
> Now the filler can branch correctly instead of just `.type()` into everything.

> ## 4. Add a “question classifier” for yes/no and special cases
> 
> Once you have `contextText`, you can write small helpers to **identify question types**.
> 
> Example: Work authorization
> 
> ```ts
> function isWorkAuthQuestion(context: string): boolean {
>   return (
>     context.includes('authorized to work') ||
>     context.includes('legally work in the united states') ||
>     context.includes('legally work in the us') ||
>     context.includes('authorization to work in the united states')
>   );
> }
> 
> function isSponsorshipQuestion(context: string): boolean {
>   return (
>     context.includes('require sponsorship') ||
>     context.includes('sponsor your employment') ||
>     context.includes('sponsorship now or in the future') ||
>     context.includes('work visa sponsorship')
>   );
> }
> ```
> 
> You can add more:
> 
> - `isRelocationQuestion`
> - `isTravelQuestion`
> - etc.
> 
> Then in `fieldMapper` or `filler`, before you fall back to the generic alias-based logic, you do:
> 
> ```ts
> if (isWorkAuthQuestion(descriptor.contextText || '')) {
>   // map to eligibility.authorized_to_work_us
> }
> 
> if (isSponsorshipQuestion(descriptor.contextText || '')) {
>   // map to eligibility.require_sponsorship_now_or_future
> }
> ```

> ## 5. Fill select/radio/checkbox properly
> 
> In `filler.ts`:
> 
> ### a) Route by kind
> 
> ```ts
> async function fillDescriptor(page: Page, desc: FieldDescriptor, profile: Profile) {
>   const ctx = desc.contextText || '';
> 
>   // Example: work authorization (yes/no)
>   if (isWorkAuthQuestion(ctx)) {
>     return fillYesNo(page, desc, profile.eligibility.authorized_to_work_us);
>   }
> 
>   if (isSponsorshipQuestion(ctx)) {
>     return fillYesNo(page, desc, !profile.eligibility.require_sponsorship_now_or_future);
>   }
> 
>   // More question types go here...
> 
>   // Otherwise generic by logical field
>   const logicalField = guessLogicalField(desc, profile.field_aliases);
>   if (!logicalField) return; // log skip
> 
>   // now branch by fieldKind
>   switch (desc.fieldKind) {
>     case 'text':
>     case 'textarea':
>       return fillTextField(page, desc.elementHandle, getValueForLogicalField(profile, logicalField));
>     case 'select':
>       return fillSelectField(page, desc.elementHandle, getValueForLogicalField(profile, logicalField));
>     case 'radio':
>     case 'checkbox':
>       return fillBooleanOrOption(page, desc, getValueForLogicalField(profile, logicalField));
>     default:
>       return;
>   }
> }
> ```
> 
> ### b) Yes/No helper
> 
> ```ts
> async function fillYesNo(page: Page, desc: FieldDescriptor, answer: boolean) {
>   const desired = answer ? 'yes' : 'no';
> 
>   if (desc.fieldKind === 'select') {
>     // choose option
>     await chooseOptionByLabel(page, desc.elementHandle, desired);
>   } else if (desc.fieldKind === 'radio' || desc.fieldKind === 'checkbox') {
>     // find the right input + label that matches 'yes' or 'no'
>     await clickYesOrNoOption(page, desc.elementHandle, desired);
>   } else {
>     // Fallback: type 'Yes' or 'No'
>     await fillTextField(page, desc.elementHandle, desired.charAt(0).toUpperCase() + desired.slice(1));
>   }
> }
> ```
> 
> Where `chooseOptionByLabel` loops through `<option>` values and chooses best match:
> 
> ```ts
> async function chooseOptionByLabel(page: Page, el: ElementHandle, desired: string) {
>   const valueToSelect = await page.evaluate((select, desiredVal) => {
>     const opts = Array.from((select as HTMLSelectElement).options);
>     const lowerDesired = desiredVal.toLowerCase();
> 
>     const exact = opts.find(o => o.textContent?.toLowerCase().includes(lowerDesired) || o.value.toLowerCase().includes(lowerDesired));
>     return exact?.value || null;
>   }, el, desired);
> 
>   if (valueToSelect) {
>     await el.select(valueToSelect);
>   }
> }
> ```

> ## 6. “Can we use computer vision?”
> 
> Short, honest answer:
> 
> - **You *could*** use literal computer vision (screenshots → OCR → LLM / classifier),  
> - But for job apps, it’s **massive overkill** and slower + more brittle than just scraping context from the DOM.
> 
> What **actually behaves like “vision”** without the pain:
> 
> - Use surrounding text: parent div, sibling `<p>`, section headers
> - Normalize all that into `contextText`
> - Run a tiny rules engine + regex + some hand-crafted logic, plus your `field_aliases`.
> 
> If you really want to go wild later, you *can* add an LLM call:
> 
> > send `contextText` and `fieldKind` to OpenAI/Gemini and say  
> > “classify this as: full_name / email / work_auth / sponsorship / relocation / other”
> 
> …but that’s “phase 2”.  
> Right now, you can fix 80–90% of the dumb behavior just by:
> 
> - Adding `eligibility` data  
> - Using `contextText`  
> - Branching by `fieldKind`  
> - Writing a few `isXQuestion()` helpers.
