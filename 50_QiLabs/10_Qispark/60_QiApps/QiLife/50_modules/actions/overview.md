# Actions: Overview
The **Actions** module serves as the primary task ledger for active execution in QiLife. In the **Personal LifeDesk** model, an Action represents a specific work order or task that Cody (the agent) must perform. Actions differ from raw QiBits because they are fully structured, estimated, and assigned to execution states.
